HOW IT WORKS
================================================================================

When a user visits the app, proxy.ts initializes Clerk auth on every request.
The root layout wraps everything in ClerkProvider so auth state is available
client-side. bible-app.tsx is the main component — it calls
useReadingProgress() which fetches the user's saved progress from
/api/progress (Neon database) on load, or falls back to localStorage if not
signed in. When a user taps a chapter, toggleChapter updates local state and
immediately POSTs the change to /api/progress which upserts it in Neon.

The home tab shows fall-challenge-view.tsx — a September calendar where tapping
a day reveals that day's chapters to mark off. The plan tab (plan-page.tsx)
shows the full schedule with all days and stats. The profile tab shows overall
completion stats. All chapter toggle buttons use the shared chapter-toggle.tsx
component. Progress is scoped to the fall-challenge plan only.

Stack: Next.js 16 + React 19 + TypeScript + Tailwind + Clerk (auth) +
Prisma 7 + Neon (Postgres) + Vercel (hosting) — all on free tiers.
